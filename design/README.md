# `design/` — visual artifacts

This folder holds **rendered diagrams** that complement `docs/ARCHITECTURE.md` and
`docs/LLD.md`. Diagrams here are **read-only artifacts**: every change to a class,
interface, or data flow MUST be reflected in source `.puml` / `.mmd` files and a
re-rendered SVG/PNG checked in alongside.

## Suggested artifacts (add as needed)

| File                                 | Purpose                                                    |
| ------------------------------------ | ---------------------------------------------------------- |
| `architecture.svg` / `.puml`         | C4-Context / container view of the engine.                  |
| `class_diagram.svg` / `.puml`        | UML class diagram for `IPathfinder`, `IHeuristic`, `Grid`. |
| `sequence_run.svg` / `.puml`         | Sequence: `App::run` → algorithm → visualizer → metrics.   |
| `sequence_benchmark.svg` / `.puml`   | Sequence: `--benchmark` mode through `Benchmark::sweep`.   |
| `state_visualizer.svg` / `.puml`     | State machine for the CLI visualizer modes.                |
| `bench_results_<map>.svg`            | Benchmark plots (generated by `scripts/perf_diff.py`).     |

## Authoring rules

1. Source files (`.puml`, `.mmd`) and rendered output (`.svg`, `.png`) live **side by
   side**. Do not commit the rendered output without its source.
2. Use the **same names** in diagrams that exist in code (`pae::algo::AStar`, not
   "AStar Class").
3. Keep diagrams **scope-bounded** — one concern per file. Cross-link in
   `docs/ARCHITECTURE.md` instead of bloating a single mega-diagram.
4. SVG is preferred over PNG for clarity at any zoom level.
